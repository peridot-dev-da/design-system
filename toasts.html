Design System / Toast Notifications

Toast Notifications

Lightweight notifications that appear temporarily to provide feedback or important information.

Basic Toasts

Toast notifications appear from the edge of the screen and auto-dismiss after a few seconds. 

// Basic toast notifications
showToast('success', 'Operation completed successfully!');
showToast('error', 'Something went wrong. Please try again.');
showToast('warning', 'This action cannot be undone.');
showToast('info', 'Here\'s some helpful information.');

// Using the underlying showAlert function
window.showAlert({
    type: 'success',
    message: 'Your changes have been saved.',
    toast: true,
    autoClose: true,
    autoCloseDelay: 5000
});

Toast Positions

Toasts can appear from different screen positions: top-right (default), top-center, top-left, bottom-right, bottom-center, or bottom-left. 

// Toast with custom position
window.showAlert({
    type: 'success',
    message: 'Toast positioned at top-center',
    toast: true,
    position: 'top-center',
    autoClose: true
});

// Available positions:
// 'top-right', 'top-center', 'top-left'
// 'bottom-right', 'bottom-center', 'bottom-left'

Toasts with Titles

Add titles to provide context and make notifications more informative. 

// Toast with title
window.showAlert({
    type: 'success',
    title: 'Upload Complete',
    message: 'Your file has been uploaded successfully.',
    toast: true,
    autoClose: true,
    autoCloseDelay: 6000
});

Toast Sizes

Toasts come in three sizes: small, default, and large to accommodate different amounts of content. 

// Toast sizes
window.showAlert({
    type: 'info',
    message: 'Small toast notification',
    toast: true,
    size: 'sm',
    autoClose: true
});

window.showAlert({
    type: 'info',
    message: 'Large toast with more content',
    toast: true,
    size: 'lg',
    autoClose: true
});

Persistent Toasts

Toasts that don't auto-dismiss, requiring user interaction to close. Useful for important notifications. 

// Persistent toast (no auto-close)
window.showAlert({
    type: 'warning',
    title: 'Important Update',
    message: 'Your session will expire in 5 minutes.',
    toast: true,
    autoClose: false,  // Don't auto-dismiss
    closable: true     // Show close button
});

In-page Alerts

Static alerts that appear within the page content, useful for form validation or persistent status messages. 

// In-page alert (not a toast)
window.showAlert({
    type: 'success',
    message: 'Your changes have been saved.',
    toast: false,      // Appears in page content
    autoClose: false,
    closable: true
});

JavaScript API Reference

Complete reference for the toast notification system API. 

// Quick notification function
window.notify(message, type);

// Examples:
notify('Task completed!', 'success');
notify('Connection lost', 'error');
notify('Changes saved', 'info');

// Full showAlert function with all options
window.showAlert({
    type: 'primary',           // primary, secondary, success, warning, error, info, neutral
    title: null,               // Optional title
    message: 'Alert message',  // Main message content
    icon: true,                // Show/hide icon
    closable: true,            // Show/hide close button
    autoClose: false,          // Auto-dismiss after delay
    autoCloseDelay: 5000,      // Delay in milliseconds
    size: 'default',           // sm, default, lg
    accent: false,             // Add accent border
    toast: false,              // True for toast, false for in-page alert
    position: 'top-right'      // Toast position
});

// Available toast positions:
// 'top-right'    - Default position (recommended)
// 'top-center'   - Top center of screen
// 'top-left'     - Top left corner
// 'bottom-right' - Bottom right corner
// 'bottom-center'- Bottom center of screen
// 'bottom-left'  - Bottom left corner

Best Practices

When to Use Toasts

  • Success notifications: Form submissions, file uploads, saved changes
  • Error messages: Failed operations, validation errors, network issues
  • Information updates: System status, feature announcements
  • Warnings: Session timeouts, destructive actions

Guidelines

  • Keep messages concise: Toasts should be scannable at a glance
  • Use appropriate timing: Success toasts can auto-dismiss quickly, errors may need longer
  • Avoid overuse: Too many toasts can overwhelm users
  • Consider context: Use in-page alerts for form validation, toasts for global feedback

Accessibility

  • Toasts automatically include appropriate ARIA attributes
  • Screen readers will announce toast content
  • Color is not the only indicator - icons and text provide context
  • Close buttons are keyboard accessible
